# 队列与引导语义说明

本文档只定义产品语义，不讨论当前代码实现。

## 目标

聊天系统里有两个很容易混淆的概念：

- 队列
- 引导

它们都和“下一条用户消息何时进入下一轮 LLM 调度”有关，但语义完全不同。

如果不先把这两个概念拆清楚，后续实现就会反复把“普通排队”误写成“引导续答”，或者把“引导插队”误写成“只是换个状态标签”。

## 一句话定义

### 队列

队列的语义是：

`等当前这一次 LLM 调度完整结束后，再把这条消息发出去。`

这里的“完整结束”指的是：

- 本轮没有继续中的工具调用
- 本轮没有继续中的正文流式输出
- 本轮已经自然收口

也就是说，队列遵守的是：

`本轮完整结束 -> 下一轮再开始`

### 引导

引导的语义是：

`当前这一次 LLM 调度只要完成了一次工具执行，就立刻打断当前调度，把引导消息插入进去，并重新发起一轮新的调度。`

这里的关键不是“等本轮彻底说完”，而是：

- 只要工具已经执行完一轮
- 就允许提前截断当前调度
- 立刻插入引导消息
- 然后重新起一轮新的 LLM 调度

也就是说，引导遵守的是：

`一次工具执行完成 -> 立刻切断 -> 插入消息 -> 重新起轮次`

## 两者的根本区别

根本区别不在于“是否高优先级”，而在于：

- 队列改变的是“下一轮何时开始”
- 引导改变的是“当前轮次在哪里被截断”

换句话说：

- 队列不改当前轮次的结束条件
- 引导直接改当前轮次的结束条件

## 队列的正确理解

队列适合处理的是“这条消息可以等”。

典型语义：

1. 当前 LLM 正在跑
2. 新消息先挂起
3. 等当前整轮自然完成
4. 再把挂起消息送入下一轮

所以队列的重点是：

- 不打断当前轮次
- 不插入当前轮次中间
- 不修改当前轮次的收口时机

## 引导的正确理解

引导适合处理的是“这条消息不能再等到整轮自然说完，而是要在工具执行完成后尽快插入”。

典型语义：

1. 当前 LLM 正在调工具
2. 某一次工具执行完成
3. 系统发现存在引导消息
4. 当前调度不再继续自然往下跑
5. 立即切断当前调度
6. 插入引导消息
7. 重新发起新一轮 LLM 调度

所以引导的重点是：

- 它不是普通排队
- 它不是“等当前轮次完整结束”
- 它依赖“工具执行完成”这个切点
- 它会改变当前调度的后续走向

## 不允许混淆的点

以下理解都是错误的：

### 错误 1：引导只是队列的一种样式

不对。

引导不是普通队列消息换个标签。

如果只是标签变化，而不改变调度切点，那它仍然只是普通队列，不是引导。

### 错误 2：引导等于优先级更高的队列

也不对。

“优先级更高”只是在多个待发消息之间决定谁先发。

但引导真正改变的是：

- 当前轮次是否继续跑完
- 当前轮次在工具完成后是否立刻截断

这不是优先级问题，而是调度边界问题。

### 错误 3：引导一定要等到正文说完

不对。

如果要等正文自然说完，那就已经退化成普通队列了。

引导的切点应该是：

`完成一次工具执行`

而不是：

`整轮自然结束`

## 推荐的判断标准

判断一条消息到底是“队列”还是“引导”，只问一个问题：

`它是否要求改变当前调度的结束时机？`

如果答案是：

- 否：它是队列
- 是：它是引导

再具体一点：

- 想等这一轮彻底结束后再说：队列
- 想在工具执行完一轮后立刻插进去：引导

## 对实现的约束

虽然本文档不规定具体代码，但语义上至少要满足这些约束：

### 队列必须满足

- 当前轮次完整结束前，不得插入
- 不得因为队列消息存在而提前截断当前轮次
- 队列的存在只影响“下一轮的输入集合”

### 引导必须满足

- 引导的存在可以改变当前轮次的收口时机
- 切断点以“完成一次工具执行”为准
- 切断后必须插入引导消息并重新发起调度
- 不能把“引导中”实现成普通排队的假象

## 一组最小例子

### 例子 1：普通队列

当前轮次：

1. LLM 调工具
2. 工具返回
3. LLM 继续思考
4. LLM 输出正文
5. 本轮完整结束

这时新来的队列消息：

- 不插入第 2 步和第 3 步之间
- 不插入第 3 步和第 4 步之间
- 不插入第 4 步和第 5 步之间
- 只能等第 5 步结束后进入下一轮

### 例子 2：引导

当前轮次：

1. LLM 调工具
2. 工具返回

这时存在引导消息，则正确行为是：

3. 立即截断当前调度
4. 插入引导消息
5. 重新发起一轮新的 LLM 调度

而不是：

3. 继续让原轮次把正文说完
4. 结束后再处理引导

后者不是引导，只是排队。

## 最终结论

队列和引导的本质区别是：

- 队列：等待当前调度完美结束后再发
- 引导：当前调度只要完成一次工具执行，就立刻打断、插入、重启调度

所以：

`队列决定“下一轮什么时候开始”，引导决定“当前轮次什么时候被截断”。`

这是后续所有实现、测试、前端文案、运行态展示都必须服从的统一定义。
