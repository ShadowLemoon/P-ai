## 记忆生成规则

### 一、核心原则
- **记忆定义**：未来可能复用的稳定信息，不是对话摘要。
- **原子化**：一条记忆只表达一个清晰论断。
- **有依据**：必须能从对话中找到明确证据，不推测、不评价。
- **高敏禁记**：身份证号、密码、验证码、密钥等绝不记录。

### 二、记忆类型
1. **AI自身记忆**：记录AI已查证的外部事实、技术结论、项目约束、工作流程等长期认知。不绑定用户画像。
2. **用户记忆**：记录具体用户的稳定信息，以用户画像形式存储（别名、属性、技能、关系、项目、要求）。

### 三、是否生成：判断流程
按顺序检查：
1. 未来可能复用吗？否 → **不记**。
2. 属于AI自身认知还是用户信息？
3. 足够稳定、非临时吗？否 → 不记。
4. 有明确对话依据吗？否 → 不记。
5. 落入禁止项（寒暄、情绪、单次操作、高敏等）吗？是 → 不记。
6. 检查旧记忆：无则新增；相似则**合并**；纠正则**修正**；避免矛盾。

### 四、禁止记录的内容
- 礼貌语、寒暄、情绪宣泄、玩笑、单次口误。
- 按钮点击、发文件等低价值元信息。
- AI自身执行过程描述。
- 短期任务步骤（非长期项目稳定阶段）。
- 用户原问题（除非可提炼为稳定结论）。
- 对用户的贬损性判断、性格推测。
- 高敏身份/财务信息（身份证、银行卡等）及任何密码、token。

### 五、记忆组成规范
每条记忆必须包含 `judgment`、`reasoning`、`tags`。

**1. judgment（论断主体）**
- 完整陈述句，直接写结论。
- 禁用“用户说过”“本轮对话”等过程词，不照抄问题。
- 不评价、不夸大、不推断。
- 涉及具体用户时，必须使用格式：`用户昵称（用户ID）`。
- 脱离当前语境也能独立理解。

**2. reasoning（依据说明）**
- 只写事实来源、对话依据或确认背景。
- 不写目的性文字（如“为了后续使用”），不写模型心理活动。
- 不是第二个 judgment，不在此隐藏新事实。

**3. tags（搜索锚点）**
- 简短可检索的词语，如：人物、用户ID、项目、工具、技能、作品、地点、组织、稳定属性等。
- 避免完整句子或细碎标签堆砌。

### 六、用户画像记忆特殊规则
当记忆对象为具体用户时，必须按以下画像规则生成。

**内容类别（画像属性，仅限以下6个中文标签）：**
- **用户别名**：稳定昵称、代号，必须绑定用户ID。
- **事实属性**：低敏长期事实（所在地、职业、身份、稳定偏好、习惯等）。
- **技能树**：已掌握、学习中或常用的技能/工具/语言/框架（不记一次性尝试）。
- **关系图谱**：用户与人、群体、AI、组织、项目、作品、地点间的稳定关系。
- **活跃项目**：正在推进的项目/作品/研究，写明推进人和当前目标。
- **用户要求**：用户明确要求后续持续遵守的约束、禁忌、回答方式等（非临时指令）。

**tags 格式（严格强制）：**
- 必须至少包含三个核心标签：`用户昵称`、`用户ID`、以及上述`画像属性`之一。
- 可适当补充其他稳定检索标签，避免堆砌。

**judgment 写法补充：**
- 必须包含“用户昵称（用户ID）”。
- reasoning 提供依据，tags 严格按上述设定。

### 七、合并与修正
- 与旧记忆相似：合并，使信息更准确、更完整、更简洁。
- 纠正旧记忆：直接修正旧记忆，不新增矛盾条目。
- 用户明确纠正画像：只修正对应画像记忆，保持昵称+ID+画像属性标签不变。

### 八、最终质量检查
生成后逐条核实：
- 未来可复用？
- 仅一个论断？
- 有依据，无过度推断？
- 无临时状态、无敏感信息？
- judgment 独立可读？
- reasoning 仅为依据，不含新事实？
- tags 均为搜索锚点？
- 若是用户画像：tags 是否至少包含昵称+ID+六个中文画像属性之一？
- 不确定时，默认不记。
