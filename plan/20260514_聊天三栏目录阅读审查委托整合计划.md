# 聊天三栏目录阅读审查委托整合计划

## 背景

当前聊天窗口已经形成左中右三栏结构，并且标题栏三列可与下方栏宽对齐。现有独立文件阅读窗口具备文件树、文件阅读、Markdown/代码渲染、拖拽打开文件等能力，但它作为完整窗口 App 存在，直接塞进聊天窗口会把窗口控制、标题栏、独立会话状态等逻辑混入聊天视图。

本计划目标是把文件浏览/阅读能力整合进聊天窗口，同时保持现有独立文件阅读窗口可用。

## 最终交互结构

左侧标题栏标签：

- `目录`：左栏显示当前会话 shell 工作目录的文件树。
- `本地`：左栏显示本地会话列表。
- `远程`：左栏显示远程联系人会话列表。

右侧标题栏标签：

- `阅读`：右栏显示完整文件阅读器，支持拖拽打开文件。
- `审查`：右栏显示审查与评估相关功能；评估是审查面板内部的子功能，不作为顶层入口。
- `委托`：右栏显示委托状态和委托管理。

侧栏开关规则：

- 左栏关闭时，左栏打开按钮显示在中间列左侧；左栏打开时，关闭按钮显示在左列。
- 右栏关闭时，右栏打开按钮显示在中间列右侧；右栏打开时，关闭按钮显示在右列。
- 左右栏关闭时不应保留无内容占位列，右列仍需保留窗口控制按钮空间。

## 设计原则

- 复用现有文件阅读能力，不重写 Markdown/代码渲染、目录读取、文件读取、拖拽打开。
- 文件阅读器核心组件不依赖窗口 label、窗口控制按钮或独立窗口标题栏。
- 顶层模式由聊天窗口管理：左栏管理 `目录/本地/远程`，右栏管理 `阅读/审查/委托`。
- 右栏内容组件只关注自身功能，不负责决定自己是否打开。
- 不移除现有工具审查/评估能力，只调整入口和层级。

## 状态模型

新增前端状态：

```ts
type ChatLeftPanelMode = "directory" | "local" | "contact";
type ChatRightPanelMode = "reader" | "review" | "delegate";
```

建议持久化：

- `easy-call.chat.left-panel-mode`
- `easy-call.chat.right-panel-mode`
- 继续沿用现有左右栏打开状态和栏宽持久化。

模式与可见性的关系：

- 点击 `目录/本地/远程`：设置 `leftPanelMode`，必要时打开左栏。
- 点击 `阅读/审查/委托`：设置 `rightPanelMode`，必要时打开右栏。
- 关闭侧栏不清空 mode，重新打开恢复上次 mode。

## 组件拆分

### 文件阅读器拆分

从 `src/apps/file-reader/FileReaderApp.vue` 抽出：

- `FileReaderPanel.vue`
  - 管理文件 tabs、active file、目录树、地址栏、阅读内容、原文/渲染切换。
  - 支持 `initialRootPath`、`initialOpenPath`。
  - 支持拖拽打开文件。
  - 暴露 `openPath(path)`，用于左侧目录树点击后打开文件。

- `FileReaderWindowApp.vue` 或保留 `FileReaderApp.vue` 作为窗口壳
  - 负责独立窗口标题栏、窗口按钮、窗口尺寸、file-reader open-path 事件。
  - 内部挂载 `FileReaderPanel`。

可复用现有内容：

- `register-file-reader-markstream.ts`
- `FileReaderCodeBlockNode.vue`
- `read_file_reader_file`
- `list_file_reader_directory`
- `open_file_reader_directory_shell`

### 左栏目录树

新增或抽出：

- `ChatWorkspaceDirectorySidebar.vue`
  - 输入当前会话 shell workspace root。
  - 使用 `list_file_reader_directory` 读取目录。
  - 点击目录：展开/收起。
  - 点击文件：通知父级打开右栏 `阅读` 并调用阅读器打开文件。

目录根路径来源：

- 优先使用当前会话主 shell 工作目录。
- 若当前会话没有自定义目录，显示空状态或系统默认说明，不静默失败。

### 右栏功能容器

新增：

- `ChatRightPanelHost.vue`
  - 根据 `rightPanelMode` 渲染：
  - `reader` -> `FileReaderPanel`
  - `review` -> 现有审查/评估面板
  - `delegate` -> 委托状态/管理面板

现有 `ToolReviewSidebar.vue` 可以保留，但需要把委托内容逐步从其中剥离到委托面板，或先以最小改动在 `delegate` 模式下复用委托状态块。

## 实施步骤

1. 抽 `FileReaderPanel`
   - 从独立窗口 App 中分离文件树、tabs、阅读内容和拖拽逻辑。
   - 独立窗口改为挂载 `FileReaderPanel`，确保原文件阅读窗口行为不变。

2. 接入左栏 `目录/本地/远程`
   - 上提左栏模式状态。
   - 标题栏左侧改为文字标签。
   - `本地/远程` 继续驱动现有会话列表过滤。
   - `目录` 显示当前会话目录树。

3. 接入右栏 `阅读/审查/委托`
   - 上提右栏模式状态。
   - 标题栏右侧显示三大标签。
   - `阅读` 渲染 `FileReaderPanel`。
   - `审查` 渲染现有审查/评估能力。
   - `委托` 渲染委托管理能力。

4. 文件树联动阅读器
   - 左侧目录点击文件后，打开右栏并切到 `阅读`。
   - 如果阅读器已经打开，新增/激活对应 tab。

5. 清理旧入口
   - 移除聊天工具条里和右栏重复的显式入口。
   - 保留独立文件阅读窗口入口。
   - 保留审查/评估组件和后端能力。

## 风险与处理

- 文件阅读器当前是窗口级 App，抽组件时容易引入行为回归。
  - 处理：先抽组件但保持独立窗口视觉与行为不变，再接聊天右栏。

- 拖拽打开文件在嵌入场景可能和聊天附件拖拽冲突。
  - 处理：阅读器右栏区域内拖拽只交给阅读器处理；聊天主区拖拽仍走附件逻辑。

- 当前会话目录为空或不存在时可能让用户误以为功能坏了。
  - 处理：目录栏必须显示明确空状态和当前目录来源。

- 审查/评估/委托目前可能耦合在同一个侧栏组件内。
  - 处理：第一阶段可保留原组件复用，第二阶段再拆细；但标题栏顶层模式先定下来。

## 验证范围

- `pnpm typecheck`
- 文件阅读独立窗口打开、拖拽、目录树、Markdown/代码渲染。
- 聊天窗口左侧 `目录/本地/远程` 切换。
- 聊天窗口右侧 `阅读/审查/委托` 切换。
- 从左侧目录点击文件，在右侧阅读器打开。
- 聊天附件拖拽与阅读器拖拽互不干扰。

## 待裁决

- `目录` 是否在点击时自动打开左栏：建议自动打开。
- `阅读/审查/委托` 是否在点击时自动打开右栏：建议自动打开。
- 独立文件阅读窗口是否继续保留托盘/命令入口：建议保留。
- 委托面板第一阶段是否允许复用现有审查侧栏内部委托块：建议允许，降低首轮风险。
