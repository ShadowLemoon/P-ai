# 20260506 部门多人格配置计划

## 背景

当前 `DepartmentConfig.agentIds` 在数据结构上已经是数组，但设置页仍按“单负责人”设计，导致：

- 用户无法在同一部门里维护多个人格
- 多人格部门的配置能力被 UI 人为锁死
- `mention` 等后续 runtime agent 设计缺少配置入口

## 目标

把部门的人格配置从“单选一个人格”升级为“一个部门可配置多个人格并维护顺序”。

本轮目标：

1. 设置页支持一个部门挂多个人格
2. 支持添加、移除、调整顺序
3. 第一位人格继续作为当前兼容阶段的主执行人格
4. 其他人格先服务于 `mention` 与后续 runtime agent 过渡

## 规则

### 1. 顺序语义

- `agentIds[0]`：当前兼容阶段的主执行人格
- 其余 `agentIds[n]`：该部门下的附加人格

### 2. 配置行为

- 可从可用人格列表中追加
- 可移除已挂载人格
- 可上移/下移调整顺序
- 不允许重复添加同一人格

### 3. 保存行为

- 继续保存到 `DepartmentConfig.agentIds`
- 助理部门保存后，仍以第一人格同步现有主助理设置

## 实现范围

- `src/features/config/views/config-tabs/DepartmentTab.vue`
- 必要时补少量文案与类型适配
- 消费侧最小支持：
  - `mention`
  - 部门树展示

## 非目标

本轮不做：

- 全部运行时按多人格并发执行
- 所有 `agentIds[0]` 读取点一次性重写
- 记忆、调度、会话索引的全链路 runtime agent 化

## 风险与约束

- 现有大量逻辑仍默认只认第一人格
- 所以本轮必须显式保留“第一人格=兼容主人格”的过渡规则

## 验证

- 设置页能为一个部门添加多个人格
- 顺序调整后保存不丢失
- 部门树能看到多个绑定人格
- 与 `mention` 联动后能出现同一部门下的多人格条目
- `pnpm typecheck` 通过
