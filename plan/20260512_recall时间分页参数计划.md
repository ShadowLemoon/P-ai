# recall 时间分页参数计划

## 背景

`recall` 当前只按查询相关性返回最多 7 条记忆，无法按更新时间浏览最近或更早的记忆，也无法分页。现在 `remember` 已支持 update/merge，模型需要更可控地查看候选记忆，避免只依赖相关性召回。

## 目标

- `recall` 增加可选 `query`、`time`、`offset`、`limit`。
- 四个参数同时生效：先按 `query` 过滤相关记忆；`query` 为空时使用全部可见记忆；再按 `time` 过滤年/月/日；然后用 `offset` 跳过结果；最后用 `limit` 控制返回数量。
- `time` 支持 `YYYY` / `YYYY-MM` / `YYYY-MM-DD`：只写年表示该年，只写到月表示该月，写到日表示该日。
- `offset` 表示跳过多少条结果，默认 0。
- `limit` 表示返回多少条结果，默认 7，做上限保护避免一次返回过多。
- 默认按记忆 `updatedAt` 优先、缺失时用 `createdAt`，从新到旧排序；本次不暴露排序参数。

最终工具入参：

```json
{
  "query": "可选的回忆查询文本",
  "time": "YYYY | YYYY-MM | YYYY-MM-DD",
  "offset": 0,
  "limit": 20
}
```

处理顺序：

1. `query` 有值时做相关性过滤；无值时使用全部可见记忆。
2. `time` 有值时按年/月/日过滤 `updatedAt` 或 `createdAt`。
3. 按时间从新到旧排序。
4. 用 `offset` 跳过指定条数。
5. 用 `limit` 返回指定条数。

## 实现范围

- 更新 `RecallToolArgs` 与 `recall` provider schema。
- 扩展 `builtin_recall` 的排序、分页和返回字段。
- 补充最小相关 Rust 单测，覆盖 `time` 格式校验。
- 更新 `CHANGELOG.md`。

## 不做事项

- 不增加 `timeOrder`，排序固定为新到旧。
- 不使用 start/end 时间范围，避免工具参数变复杂。
- 不要求 `query` 必填。

## 确认

用户已确认修改 recall 支持按时间、offset、limit。
