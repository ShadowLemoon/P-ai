# 会话列表实时运行状态通知计划

## 背景

当前会话列表第二排主要显示最近消息预览，运行状态只在右侧以较弱文字展示。对于多会话并行、后台会话仍在工作、工具调用或失败重试的场景，用户无法一眼判断“哪个会话正在动、现在动到哪一步”。

用户希望会话列表成为轻量实时状态面板：不需要复杂的第二排文案，只要运行中有明确转圈提示，完成有成功态，失败有失败态。完成态代表“后台会话有未查看的完成结果”，应保留到用户切入该会话；失败状态切回会话后仍应持续显示，直到后续状态覆盖或用户手动清理错误提示。

## 目标

- 会话列表保留第二排最近消息预览，不引入复杂状态文案。
- 会话忙碌时，在未读/状态区域显示持续旋转状态。
- 会话完成时持续显示成功态，直到用户切入该会话。
- 会话失败时显示失败态，并且切回该会话后不自动清除。
- 聊天错误横幅提供关闭按钮，由用户显式清理。
- 后台完成追加新消息时，未读数即时增加，不等待概览刷新。
- 本地会话列表 `ChatConversationSidebar` 和弹出会话列表 `ChatConversationListCard` 逻辑保持一致。

## 非目标

- 不改变后端消息存储结构。
- 不把运行态写入持久化会话数据。
- 不增加新的重型全局状态库。
- 不改变聊天主面板的状态横幅逻辑。

## 当前可复用能力

- `usePipelineStatus()` 已监听 `conversation_work_status`，能拿到每个会话 `busy/error` 粗状态。
- `ChatConversationOverviewItem.runtimeState` 已支持 `assistant_streaming` / `organizing_context`。
- `useChatFlow` 内已有当前会话 `frontendRoundPhase`、`toolStatusText`、`toolStatusState`，可用于当前会话精细状态。
- `UnifiedWindowApp.vue` 已维护 `toolStatusText`、当前会话 id、会话列表 items，可在前端合成临时 UI 状态。

## 方案

### 1. 新增会话列表 UI 状态模型

新增轻量前端类型，例如：

```ts
type ConversationPipelineStatus = "busy" | "success" | "error";
```

该状态只用于渲染，不持久化。

### 2. 运行状态来源

复用 `usePipelineStatus()` 监听 `conversation_work_status`：

- `busy`：显示“正在处理”
- `success`：显示未查看完成态，保留到用户切入该会话
- `error`：显示“处理失败”

成功状态由 `markConversationRead()` 在用户切入会话后清理。失败状态不因 `markConversationRead()` 自动清除，避免切回会话后失败提示消失；用户关闭聊天错误横幅时同步清理当前会话的失败状态。

### 3. 列表渲染

- 第二排继续显示最新消息预览。
- 右侧徽标区域：
  - `busy` 显示 spinner。
  - `success` 在头像上显示成功点。
  - `error` 显示失败 badge。
  - 无 pipeline 状态时回退显示原有 runtimeState / 未读数。

后台会话收到正式助手消息时，本地即时更新会话预览、消息数和未读数，避免必须等待下一次概览刷新才出现未读徽标。

### 4. 错误横幅

聊天窗口顶部错误横幅新增关闭按钮。点击后清空当前会话的 `chatErrorText`，同步清理会话列表中当前会话的失败态，并在当前工具状态为 `failed` 时同步清理当前工具失败态。

### 5. 组件覆盖范围

同步改：

- `ChatConversationSidebar.vue`
- `ChatConversationListCard.vue`

避免左右两套会话列表表现不一致。

## 验证

- `pnpm typecheck`
- 手动验证当前会话：
  - 发送后列表项显示 spinner
  - 完成后显示成功态，直到切入该会话
  - 失败后显示失败态
  - 切回失败会话后失败态仍持续显示
  - 错误横幅关闭按钮可清理当前错误
- 手动验证后台会话：
  - 后台 busy 时列表项转圈
  - 完成后立即显示成功态和未读数
  - error 时显示失败态
  - 切入会话后状态清理正常

## 风险

- 第一版只显示粗状态，不显示具体工具名；这是刻意收窄范围，避免列表变成复杂状态文本面板。
- 成功态和失败态都是运行期 UI 状态，不持久化到磁盘；刷新后依赖后端概览和运行态重新同步。
